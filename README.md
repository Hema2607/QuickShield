# 🛡️ AI-Powered Parametric Insurance for Gig Workers

## 🚀 Overview

This project presents an AI-powered parametric insurance platform designed specifically for quick-commerce delivery partners such as Blinkit and Zepto. It protects riders from income loss caused by external disruptions such as extreme weather, pollution, and demand fluctuations.

Unlike traditional insurance systems, this solution provides instant, automated payouts based on predefined triggers without requiring manual claims.

---

## 🎯 Problem Statement

Gig workers depend on daily or weekly earnings. External disruptions such as:

* Heavy rain  
* Extreme heat  
* High pollution  
* Sudden drop in demand  

can reduce their working hours, causing up to 20–30% income loss, with no financial safety net.

---

## 💡 Proposed Solution

We propose a hyperlocal, time-based parametric insurance system that combines:

* Real-time environmental monitoring  
* AI-based risk prediction and trust scoring
* Automated trigger detection  
* Instant payout processing  

---

## ⚙️ Features

### 🧠 AI-Based Risk Assessment

* Predicts disruption probability  
* Calculates dynamic weekly premium based on location, weather patterns, and demand volatility  

---

### ⏱️ Time-Based Protection

* Users select active working hours such as peak delivery slots  
* Coverage applies only during selected time periods  
* Ensures realistic and cost-efficient insurance  

---

### 📍 Hyperlocal Risk Zones

* City is divided into zones such as OMR, T Nagar, Velachery  
* Each zone has a unique risk profile  
* Premium adjusts based on zone-specific risks  

---

### 💰 Fixed Parametric Payout System

* Predefined payout amounts based on disruption triggers  
* No income calculation required  
* Ensures fast, transparent, and reliable compensation  

Example:

Rainfall above threshold results in payout of 200 rupees  
AQI above 400 results in payout of 150 rupees  
Demand drop results in payout of 100 rupees  

---

### ⚡ Automated Claim System

* Real-time trigger detection  
* Zero manual intervention  
* Instant payout simulation  

---

### 🚨 Fraud Detection

* GPS validation  
* Activity monitoring  
* Anomaly detection  
* Trust score for workers  

---

### 📊 Dashboard

* Worker: coverage, earnings protected, claims history  
* Admin: analytics, claim patterns, trust score of workers  

---

## 🔄 Workflow

1. User registers and selects work zone and active working hours  
2. System performs risk analysis  
3. Weekly policy is generated  
4. External data is continuously monitored  
5. Disruption triggers claim automatically  
6. Payout is processed instantly  

---

## 📅 Weekly Pricing Model

```
Weekly Premium = Base + Risk Adjustment

For an example:
Base = ₹50
+ ₹10 (High rainfall risk)
+ ₹5 (High pollution)
+ ₹5 (Demand volatility)
- ₹5 (High trust score)
```
